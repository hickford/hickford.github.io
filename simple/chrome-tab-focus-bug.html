Chrome tab focus bugs

Here is an example of a couple of very frustrating bugs in current [2011] versions of Google Chrome. The two links in the list below (to the websites of competing browsers Firefox and Opera) share a common named target (see the source)

First bug. Middle-click either link. It opens in the foreground, rather than the background as expected (compare with the standard links further below if you like). This is Bug 62319 in the Chrome bug tracker which has been confirmed by a Chrome developer but there is no progress towards fixing it.

Second bug (read all the instructions first). Middle-click the first link (Firefox), the new tab is focussed (bug above). Leave the tab open and switch back to this page. Middle-click the second link (Opera). Spuriously, the Firefox tab is focussed! This is bug 84514 on the Chrome bug tracker and has yet to receive any developer attention.

  1. Firefox
  2. Opera

If you observe these bugs, I implore you to visit the bugs on the Chrome bug tracker and star them to raise their profile. You'll be asked to sign into a Google account.

Progress towards fixing the bugs

The chrome devs are unsure if the bug is in Chrome or WebKit. A WebKit developer cannot observe the bug in Safari. Webkit bug 61167.
